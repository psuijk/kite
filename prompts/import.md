# Import Discussion Extract Workflow

## Purpose

Import topics from a learning discussion held in another LLM chat. The user pastes a structured YAML block generated by the prompt in `prompts/discussion-extract.md`.

## Recognizing the Input

The user will paste a YAML block with the header `# DISCUSSION EXTRACT` containing:
- `parent_topic`: the main topic ID discussed
- `date`: when the discussion happened
- `summary`: what was covered
- `topics`: list of subtopics with `id`, `name`, `difficulty`, `description`, `depends_on`, `my_level`, `notes`

## Steps

### 1. Parse and Match

- Parse the YAML block from the user's message.
- Read `knowledge-graph.yml` and load all existing node IDs, names, descriptions, and children.
- **Match every topic in the extract against existing nodes** — check both the extract's `id` and `name` against existing node IDs, names, children IDs, and descriptions. Use semantic similarity, not just exact matching. Examples:
  - `generics_and_constraints` → matches `typescript_generics` (same concept)
  - `mapped_types_and_modifiers` → matches `mapped_types` (child of `typescript_type_system`)
  - `type_narrowing_and_unions` → matches `union_types` (child of `typescript_type_system`)
  - `typescript_mastery` (in depends_on) → matches `typescript`
- Also match `parent_topic` and all `depends_on` references against existing nodes.
- Classify each extracted topic as:
  - **Match**: maps to an existing node → update skill level only
  - **New**: no match found → will be added as a new node
- Rewrite `depends_on` references to use the matched existing node IDs.

### 2. Present for Review

Show a clear summary with your matching decisions:

```
Discussion: [summary]
Parent topic: [parent_topic name]
Date: [date]

Matched to existing nodes (will update skill level):
  1. generics_and_constraints → typescript_generics — Level update to 3
  2. mapped_types_and_modifiers → mapped_types — Level update to 4

New topics to add:
  3. satisfies_operator (difficulty: 3) — description
     Your level: 4 — notes
  4. template_literal_types (difficulty: 4) — description
     Your level: 3 — notes
```

Ask the user to confirm. They can accept all, pick specific ones, correct any matches, or modify details.

### 3. Add Confirmed Topics

For each confirmed new topic:

1. **Add to `knowledge-graph.yml`** as a new node:
   ```yaml
   topic_id:
     name: Topic Name
     type: concept
     priority: <inherit from parent topic>
     difficulty: <from extract>
     description: <from extract>
     depends_on: <from extract, validated>
     children: []
     related: []
   ```

2. **Add the topic ID to the parent's `children` list** (if not already there).

3. **Record skill level in `skills.yaml`**:
   ```yaml
   topic_id:
     level: <my_level from extract>
     confidence: 0.5
     last_tested: <date from extract>
   ```
   Use confidence 0.5 since the level is estimated from discussion, not a formal quiz.

4. **Append to `learning-log.md`**:
   ```
   - **topic_name**: Level 0 -> <my_level> (confidence: 0.50)
     Notes: Imported from discussion extract. <notes from extract>
   ```

### 4. Handle Existing Topics

For topics that already exist in the graph:
- Only update `skills.yaml` if the extracted `my_level` is higher than the current level.
- Use confidence 0.5 for discussion-based assessments.
- Log the update in `learning-log.md`.

### 5. Confirm Completion

Show a summary of what was added/updated and suggest next steps:
- "Take a quiz on [topic] to get a formal assessment"
- "Expand [topic] further"
- "See your updated status"
